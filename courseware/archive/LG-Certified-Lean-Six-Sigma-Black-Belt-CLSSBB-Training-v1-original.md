# Learner Guide - Certified Lean Six Sigma Black Belt (CLSSBB) Training

## Course Overview

This learner guide supports a practical Certified Lean Six Sigma Black Belt course. It is designed for learners who will lead improvement projects, coach Green Belts, analyze process performance, and make data-backed recommendations.

The labs follow the DMAIC structure:

1. Define the business problem and customer requirements.
2. Measure current process performance.
3. Analyze root causes and validate drivers.
4. Improve the process with tested solutions.
5. Control the improved process so gains are sustained.

## Before You Start

### Recommended Project Scenario

Use one realistic process throughout the course. Examples:

- Course registration processing time.
- Invoice approval cycle time.
- Customer support ticket resolution time.
- Defect rate in document submission.
- Late delivery rate in a service process.
- Rework rate in a manufacturing or operations process.

## Lab 01 Guide - Project Charter, VOC, and CTQ Tree

### Objectives

- Define the improvement opportunity.
- Write a strong problem statement and goal statement.
- Identify customers, stakeholders, and project roles.
- Translate VOC into measurable CTQs.

### Steps

1. Select a process for the course project.
2. Write the business case in one short paragraph.
3. Define the problem using facts, not assumed causes.
4. Write a goal statement with measurable target and timeline.
5. Identify project scope, out-of-scope boundaries, and constraints.
6. List stakeholders and assign RACI roles.
7. Capture VOC from complaints, interviews, surveys, audits, or service logs.
8. Convert broad customer needs into CTQ requirements.
9. Define each CTQ with unit, target, defect definition, and data source.
10. Review the charter for vague language and remove unsupported claims.

### Deliverables

- Project charter.
- VOC table.
- CTQ tree.
- Stakeholder/RACI list.

### Checkpoint

You can explain why the project matters, who owns it, what success means, and how success will be measured.

## Lab 02 Guide - SIPOC, Process Map, and Waste Analysis

### Objectives

- Define process boundaries.
- Build a SIPOC.
- Create a high-level process map.
- Identify waste and non-value-added work.

### Steps

1. Write the process start and end points.
2. Identify suppliers, inputs, process steps, outputs, and customers.
3. Create a SIPOC table.
4. Build a high-level process map with 6 to 10 major steps.
5. Mark decision points, handoffs, queues, and rework loops.
6. Classify each step as value-added, business-value-added, or non-value-added.
7. Identify examples of overproduction, waiting, transport, overprocessing, inventory, motion, defects, and underused talent.
8. Update project scope if the map reveals unclear boundaries.
9. Identify where data should be collected in Lab 03.

### Deliverables

- SIPOC.
- Process map.
- Waste analysis table.
- Updated scope notes.

### Checkpoint

You can explain the process boundary and show where the suspected performance issue appears.

## Lab 03 Guide - Data Collection, MSA, and Process Capability

### Objectives

- Create a data collection plan.
- Define defects and opportunities.
- Assess measurement system quality.
- Calculate baseline capability.

### Steps

1. Select the primary Y metric from the CTQ tree.
2. Write an operational definition for the metric.
3. Define defect, unit, opportunity, and sampling approach.
4. Build a data collection plan with who, what, when, where, and how.
5. Collect or use sample baseline data.
6. Identify measurement risks such as inconsistent judgment, poor calibration, or missing timestamps.
7. For continuous data, outline a Gage R&R approach.
8. For attribute data, outline an attribute agreement approach.
9. Calculate DPU, DPMO, first pass yield, rolled throughput yield, or sigma level as applicable.
10. For continuous data, calculate Cp, Cpk, Pp, or Ppk if specification limits are known.
11. Document whether the baseline is credible enough for decision making.

### Deliverables

- Data collection plan.
- Operational definition.
- MSA notes.
- Baseline capability worksheet.

### Checkpoint

You can explain whether the data can be trusted and what the current process performance is.

## Lab 04 Guide - Baseline Analysis and Control Charts

### Objectives

- Summarize baseline data.
- Visualize patterns and variation.
- Select suitable control charts.
- Distinguish common cause and special cause variation.

### Steps

1. Import baseline data into a spreadsheet.
2. Calculate count, mean, median, range, standard deviation, min, max, and quartiles.
3. Create a histogram.
4. Create a run chart over time.
5. Check whether the data appears stable before capability claims are made.
6. Select an I-MR chart for individual continuous observations when appropriate.
7. Select Xbar-R or Xbar-S charts for subgrouped continuous data when appropriate.
8. Select p, np, c, or u charts for attribute data when appropriate.
9. Plot the chart and identify possible special cause signals.
10. Annotate known events, policy changes, or seasonal periods.
11. Decide whether more data is needed before moving to Analyze.

### Deliverables

- Descriptive statistics table.
- Histogram.
- Run chart.
- Control chart.
- Baseline interpretation.

### Checkpoint

You can explain what the baseline shows and whether the process is stable enough to analyze.

## Lab 05 Guide - Root Cause Analysis and Hypothesis Testing

### Objectives

- Generate and prioritize possible root causes.
- Use Pareto and cause-and-effect tools.
- Select suitable hypothesis tests.
- Interpret statistical and practical significance.

### Steps

1. Create a fishbone diagram for the project problem.
2. Use categories such as people, process, equipment, materials, measurement, environment, and management.
3. Apply 5 Whys to at least two suspected causes.
4. Create a Pareto chart if defect category data is available.
5. Convert suspected causes into testable hypotheses.
6. Identify data type: continuous, attribute, normal, non-normal, paired, or independent.
7. Select a test such as t-test, ANOVA, proportion test, chi-square, Mann-Whitney, or Kruskal-Wallis.
8. State null and alternative hypotheses.
9. Set alpha and document the practical decision threshold.
10. Run the test using a spreadsheet or statistical tool.
11. Interpret p-value, confidence interval, effect size, and business meaning.
12. Decide which causes remain validated, unproven, or rejected.

### Deliverables

- Fishbone diagram.
- 5 Whys worksheet.
- Pareto chart if applicable.
- Hypothesis test log.
- Validated root cause list.

### Checkpoint

You can defend root causes with data instead of opinion.

## Lab 06 Guide - Regression, ANOVA, and Design of Experiments

### Objectives

- Analyze relationships between process inputs and outputs.
- Interpret regression and ANOVA results.
- Plan a designed experiment.
- Recognize experiment risks.

### Steps

1. Identify possible X variables from the root cause list.
2. Create scatter plots for continuous X and Y pairs.
3. Calculate correlation where appropriate.
4. Run simple linear regression for one likely driver.
5. Interpret coefficient, R-squared, residuals, and p-value.
6. Compare groups using ANOVA when the X is categorical and Y is continuous.
7. Check assumptions before trusting the result.
8. Identify two to four controllable factors for an experiment.
9. Define factor levels and response metric.
10. Draft a full factorial or fractional factorial experiment plan.
11. Identify randomization, replication, blocking, and safety considerations.
12. Decide whether the experiment is practical, ethical, and affordable.

### Deliverables

- Scatter plot or group comparison chart.
- Regression or ANOVA interpretation.
- DOE planning worksheet.
- Experiment risk notes.

### Checkpoint

You can explain when to use regression, ANOVA, and DOE to validate improvement levers.

## Lab 07 Guide - Improvement Selection, FMEA, and Pilot Plan

### Objectives

- Generate improvement ideas.
- Select solutions using evidence.
- Assess implementation risk with FMEA.
- Plan a controlled pilot.

### Steps

1. Brainstorm solutions linked to validated root causes.
2. Remove solutions that do not address a proven driver.
3. Score solutions by impact, effort, cost, time, and risk.
4. Select a shortlist for pilot testing.
5. Create an FMEA for the proposed future-state process.
6. Identify failure modes, effects, causes, current controls, and recommended actions.
7. Score severity, occurrence, and detection.
8. Calculate RPN or use an action priority method.
9. Define mistake-proofing and visual management options.
10. Write a pilot plan with scope, participants, duration, success criteria, and rollback plan.
11. Define data to collect during the pilot.
12. Prepare stakeholder communication for the pilot.

### Deliverables

- Solution selection matrix.
- FMEA.
- Pilot plan.
- Communication notes.

### Checkpoint

You can show why the selected solution is linked to root cause evidence and how pilot risk will be controlled.

## Lab 08 Guide - Control Plan, SPC, and Benefits Realization

### Objectives

- Build a control plan.
- Define response plans and ownership.
- Sustain gains with SPC and lean controls.
- Prepare project closure and exam review.

### Steps

1. List the improved process steps that require monitoring.
2. Define each critical input, output, control method, frequency, owner, and reaction plan.
3. Select the appropriate ongoing control chart.
4. Define escalation criteria for out-of-control signals.
5. Add lean controls such as 5S, visual management, Kanban, standard work, or poka-yoke.
6. Define handover requirements for the process owner.
7. Calculate expected financial or operational benefits.
8. Separate hard savings, soft savings, avoidance, and productivity improvement.
9. Create a benefits tracking schedule.
10. Prepare a final project story using Define, Measure, Analyze, Improve, Control.
11. Review common exam topics: CTQ, DPMO, MSA, capability, hypothesis tests, DOE, FMEA, SPC, and control plans.
12. Complete the project closure checklist.

### Deliverables

- Control plan.
- Response plan.
- Benefits realization worksheet.
- Final project storyboard.

### Checkpoint

You can explain how the improvement will be sustained after the project team leaves.
